Set up the `.claude-local/` directory for personal, untracked project files.

## Instructions

### Step 1: Check if already initialized + create directory

Run `ls -la .claude-local/ 2>/dev/null` to check existence.

If the folder already exists with files, inform the user it's already set up, show contents, and stop.

Otherwise, run `mkdir -p .claude-local`.

### Step 2: Create all files and check .gitignore IN PARALLEL

**Do all of these in a single parallel tool call batch** — there are no dependencies between them:

**Write `.claude-local/STANDUP.md`:**
```markdown
# Current Period
**Started**: Not yet initialized

## In Progress
<!-- Synced from active work -->

## Completed
<!-- Auto-populated by /commit, /complete, /pre-merge -->

## Blockers
<!-- Updated when starting new period via /standup -->

---

# Previous Period
<!-- Archived when starting new period -->
```

**Write `.claude-local/NOTES.md`:**
```markdown
# Personal Notes

Scratch space for debugging notes, research findings, and thoughts that don't belong in committed docs.

---

<!-- Add notes below -->
```

**Write `.claude-local/TODO.md`:**
```markdown
# Personal TODO

Personal task tracking separate from project tickets. Use for:
- Ideas to explore later
- Technical debt you noticed
- Follow-up items

---

## High Priority

## Normal

## Low Priority / Ideas
```

**Write `.claude-local/HANDOFF.md`:**
```markdown
# Session Handoff

No active handoff context. This file is auto-generated by `/handoff` to preserve session context.

---

*Run `/handoff` at the end of a session to populate this file.*
```

**Write `.claude-local/IMPLEMENT_STATE.md`:**
```markdown
# Implementation State

No active implementation. This file is auto-managed by `/implement` to track phase progress across sub-agents.

---

*Run `/implement` after plan approval to begin automated implementation.*
```

**Grep `.gitignore` for `.claude-local`** to check if it's already listed.

### Step 3: Update .gitignore if needed

If `.claude-local/` is NOT already in `.gitignore`, append:
```
# Claude Code local files (personal, not committed)
.claude-local/
```

### Step 4: Confirm setup

Output:
```
✅ .claude-local/ initialized

Created:
- STANDUP.md         - Work tracking for standups
- NOTES.md           - Personal debugging notes
- TODO.md            - Personal task ideas
- HANDOFF.md         - Session context for continuity
- IMPLEMENT_STATE.md - Phase tracking for /implement orchestrator

Run /standup to start tracking your work session.
```
